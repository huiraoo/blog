技术博客

为什么写技术博客如此重要 ？

写一个博客有很多好处，却没有任何明显的坏处。至少有以下好处：

  • 整理经验和分享。
  • 能把事情说清楚、讲明白。
  • 书写为了更好的思考。
  • “教”是最好的“学”。

博客数量不在多在精（每周2～3篇），数字化的时代，一切皆可数字化。

架构师

经验是架构师最宝贵的财富。

  • 业务理解能力

  • 抽象能力

  • 技术的全面性和广度：决定了架构师处理问题的效率

  • 决策能力

  • 写代码的能力：技术人安身立命的根本，读源代码是程序员成长最快的手段

技术路线发展的两个选择：

1）某技术领域的资深技术专家

2）解决问题的架构师（a. 理论基础扎实；b. 交付厉害的代码，让别人能够感受代码里的设计和思考）

工程师 - 高级工程师 - 技术专家 - 初级架构师 - 中级架构师 - 高级架构师

积累经验、拓宽视野、深度思考

工程师阶段：基础技能积累阶段（基础知识、编程语言、编程工具）

高级工程师：积累方案设计经验并能独立完成开发（需求设计、方案设计、编码实现）

技术专家：拓展技术宽度

初级架构师：能够独立完成一个系统的架构设计

中级架构师：能够完成复杂系统的架构设计

高级架构师：创造新的架构模式

软件发布全流程

  • 需求分析（需求梳理 => 产品定义）
  • 系统设计（子系统划分 => 模块定义）
  • 模块设计（模块详细设计）
  • 编码实现
  • 单元测试
  • 代码评审
  • 集成测试
  • 灰度发布
  • 正式发布
