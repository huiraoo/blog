背景

生产环境有一张日志表 schedule_task_error_log ，数据量超过 5000w，想通过 drop 删除但是速度很慢。

原理

前提：MySQL 开启了独立表空间，MySQL 5.6.7 之后默认开启。

MySQL 在删除大表的时候为什么慢？

  • Buffer Pool 页面清除过程（拥有 InnoDB 全局锁）
  • 删除 ibd 磁盘文件过程

方案：利用了 Linux 中 硬链接 减少 DDL 时间，加快锁释放，来进行快速删除。
